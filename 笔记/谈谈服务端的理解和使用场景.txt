谈谈服务端的理解和使用场景 mua

服务端的理解：
将vue实例渲染为HTML字符串直接返回，在前端激活为交互程序

使用场景
大型电商类网站首页

构建流程
目标是生成一个「服务器 bundle」用于服务端首屏渲染，和一个「客户端bundle」用于客户端激活

服务器端知识
1.首先需要搭建基础http服务
2.使用vsr
3.整合express
4.激活

备注：
npm i express -S
npm i vue vue-server-renderer -S
npm i webpack-node-externals@1.7.2 lodash.merge -D
npm i cross-env -D

优点：
1. 有利于SEO
搜索引擎的主要两个工作：爬取网站并建立索引，当用户搜索某个关键字以排名的方式将网站相关呈现给用户。自然服务器对SEO很友好
2.首屏渲染时间快
缺点：
1.高并发，负载大
如果渲染全部交给服务端处理，在高并发的情况下，对于服务端负载压力大 (更⼤的服务器端负载)
2.复杂度
前后端耦合，互相依赖 (较⾼的学习成本)
3.依赖库支持
需要依赖库支持，⼀些外部扩展库使⽤会受限

解决方案
缓存： node server
降级：cpu 内存 SPA

场景
seo，首屏，必要性，pre-renderer
